# ADR 0006 — LLM summary on the receipt

**Status:** accepted (Phase 5) · part of [[index]]

## Context

`/receipt` is the end-to-end test of the whole brief: stats + crab logo
+ project header + serial + a 3-5 line **narrative** summary that
makes the receipt feel less like a dashboard and more like a
celebration. The stats are deterministic and sacred (ADR 0005). The
narrative is generated by Anthropic Haiku and is **fail-graceful**:
any failure leaves the stats intact and prints
``(summary unavailable)`` instead of the lines.

## Decision

### Model

Pinned model id: **`claude-haiku-4-5-20251001`**. Not the marketing
label "Haiku 4.5" — the marketing label drifts across model versions
(Haiku 4.5 today, Haiku 5 next year) and pinning the id keeps prompt
behavior stable until I deliberately upgrade.

### Hard request deadline

**`REQUEST_DEADLINE_S = 10.0`**. Passed as the SDK `timeout` parameter
and treated as the wall-clock cap from invocation to either a result
or a graceful `None`. `/receipt` should print paper inside this window
or print without the summary — never block waiting.

### Transcript-excerpt slicing rule

**Last 3 user turns + last assistant turn, joined, capped at 8000
characters with tail-bias** (when the joined text exceeds the cap,
keep the most recent characters). Implemented in
`llm.slice_transcript`. Reasoning:

- Long sessions easily exceed Haiku's context if we sent the whole
  thing; the slicing rule keeps the LLM grounded in the recent moment
  without blowing the context budget.
- Tail-bias (keep the end) because the receipt is celebrating *what
  just happened*, not the whole arc.
- 3 user turns is enough to anchor the assistant's response in a
  conversation, not just a single isolated exchange.

### Fail-graceful matrix

Every one of the following returns `None` from `llm.summarize()`:

| Fault                                      | Test instrument value       |
|--------------------------------------------|-----------------------------|
| `ANTHROPIC_API_KEY` unset                  | (env var absent)            |
| `ANTHROPIC_API_KEY` unset (simulated)      | `THERMAL_PRINT_LLM_FAULT=no-api-key` |
| Request timeout / deadline exceeded        | `THERMAL_PRINT_LLM_FAULT=timeout`    |
| HTTP 401 (revoked key)                     | `THERMAL_PRINT_LLM_FAULT=401`        |
| HTTP 429 (rate-limit)                      | `THERMAL_PRINT_LLM_FAULT=429`        |
| HTTP 500 (server error)                    | `THERMAL_PRINT_LLM_FAULT=500`        |
| Malformed response body                    | `THERMAL_PRINT_LLM_FAULT=malformed`  |
| Any unexpected exception                   | (`test_summarize_never_raises`)      |

The receipt template, on `summary is None`, prints exactly
``(summary unavailable)`` and emits the rest of the layout unchanged.

### Trust boundary (carried, not new)

The transcript excerpt is **already** being sent to Anthropic via the
live Claude Code session — `/receipt` re-sending it to summarize is a
trust **extension**, not a new exposure. Worth stating explicitly so
future-Matt doesn't have to re-reason about it on every read.

### Slash command shim

`.claude/commands/receipt.md` is a minimal Bash command that runs:

```bash
thermal-print print receipt --session-id "$CLAUDE_CODE_SESSION_ID" --cwd "$PWD"
```

The session id is exposed by Claude Code in
`CLAUDE_CODE_SESSION_ID` (probed and verified on 2026-05-26; recorded
in [[build-log]]). Argv, not shell interpolation, carries the
session id and cwd — a malicious cwd cannot inject a command.

## Why

- **The receipt is the contract; the summary is a gift.** Anything
  that risks not printing paper has to lose. Every fault path was
  explicitly tested.
- **Argv over shell.** The CLI is invoked as `thermal-print print
  receipt --session-id … --cwd …` — `cwd` and `session_id` are *data*
  to the CLI, never interpreted by shell.
- **Pin the id, not the label.** A breaking prompt-behavior change
  between Haiku 4.5 and Haiku 5 would silently happen if we tracked
  the label; pinning the id makes the upgrade a deliberate one-line PR.

## Alternatives considered

- **Default to streaming.** Streaming would shave time-to-first-byte
  but adds reconnection / partial-response complexity for a 3-5 line
  generation. Rejected — the deadline is already small.
- **Bigger models (Sonnet / Opus).** Slower, more expensive, no
  qualitative win on a 5-line summary task. Rejected.
- **Send the whole transcript.** Trivially overflows Haiku's context
  on any non-trivial session. Rejected — see slicing rule.

## Consequences

- Adding `anthropic==0.104.1` as a runtime dep. README documents
  setting `ANTHROPIC_API_KEY` for the summary path to activate; the
  receipt still prints without it.
- `THERMAL_PRINT_LLM_FAULT` env var is a **test instrument** — it's
  documented in `llm.py`'s docstring; if it ever leaks into production
  use it just degrades the summary, never the receipt.
- Future model upgrade is a one-line change to `MODEL` in `llm.py`;
  the prompt and slicing rule are designed for Haiku-class models and
  should port forward.
